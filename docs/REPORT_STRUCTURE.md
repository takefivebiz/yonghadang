# VEIL 결과 리포트 구조

## 1. 문서 목적

이 문서는 VEIL 결과 리포트가 어떤 구조와 문체로 작성되어야 하는지 정의한다.
프론트엔드 UI 명세가 아니며, 프롬프트 구현 문서도 아니다.

VEIL의 결과 리포트는 AI 상담 답변, 조언형 칼럼, 일기 요약, 감성 에세이가 아니라
사용자의 감정 움직임을 구조적으로 읽어내는 심리탐정 보고서에 가까워야 한다.

## 2. 리포트 철학

VEIL은 사용자의 입력을 하나의 감정 의뢰 파일로 다룬다. 리포트의 목적은 사용자의
말을 반복하거나, 관계의 정답을 내려주거나, 즉각적인 해결책을 제시하는 것이 아니다.
입력 안에서 드러나는 감정의 움직임을 관찰하고, 이를 장면 단위의 기록으로 정리하는
것이 핵심이다.

기본 원칙:

- 외부 사건보다 감정의 움직임을 기록한다.
- 위로나 조언보다 관찰과 추리를 우선한다.
- 사용자의 표현은 그 자체가 의미를 가질 때만 제한적으로 활용한다.
- 입력이 짧아도 상태와 방향을 읽되, 확정적으로 단정하지 않는다.
- 심리탐정 톤은 조용하고 절제된 기준으로 유지한다.
- 의뢰 파일 은유는 구조를 잡기 위한 장치로만 사용한다.

## 3. 표준 리포트 구조

결과 리포트는 아래 6개 장면 구조를 기준으로 한다.

1. 01 의뢰 접수
2. 02 현장 기록
3. 03 단서 분석
4. 04 상황 추리
5. 05 핵심 추리
6. 06 최종 보고

이 구조는 현재 결과 페이지의 scene progression UX와 대응된다. 각 장면은 하나의 긴
글 안에 포함된 단락이 아니라, 사용자가 순서대로 열람하는 별도 기록처럼 느껴져야
한다.

## 4. 장면별 정의

### 01 의뢰 접수

역할:
사용자가 남긴 기록의 감정적 전제를 확인한다. 이 장면은 리포트가 사용자의 출발점을
정확히 잡고 있다는 느낌을 주어야 한다.

입력에서 읽어야 하는 것:

- 사용자가 무엇을 확인하고 싶어 하는지
- 사용자가 기록을 남기게 된 감정적 긴장
- 처음부터 드러나는 모순, 불확실성, 흔들림

출력 목표:

- 리포트 전체의 중심 질문을 설정한다.
- 사용자의 현재 상태를 판단하지 않고 명명한다.
- 가깝지만 과하게 다정하지 않은 거리감으로 파일을 연다.

금지:

- 사건 전체를 처음부터 과도하게 설명하는 것
- 바로 조언을 제시하는 것
- 따뜻한 상담 도입처럼 보이게 만드는 것

### 02 현장 기록

역할:
겉으로 드러난 감정 상황을 정리한다. 사용자의 기록에서 실제로 관찰되는 표면의
움직임을 다루는 장면이다.

입력에서 읽어야 하는 것:

- 반복되는 행동 패턴
- 사용자가 계속 확인하고 있는 신호
- 사용자가 보고 있는 것과 보지 않으려는 것

출력 목표:

- 흩어진 내용을 하나의 읽을 수 있는 감정 장면으로 정리한다.
- 사용자가 상황 안에서 어떤 위치에 있는지 보여준다.
- 첫 질문에서 더 깊은 분석으로 자연스럽게 이어지게 한다.

금지:

- 사실을 기계적으로 나열하는 것
- 사용자의 입력을 요약문처럼 다시 말하는 것
- 채팅 답변처럼 보이게 만드는 것

### 03 단서 분석

역할:
감정의 단서를 관찰자 시점에서 분석한다. 여섯 장면 중 가장 거리가 있고, 가장
조사자에 가까운 톤을 가져야 한다.

입력에서 읽어야 하는 것:

- 숨은 전제
- 반복, 망설임, 모순, 강도
- 사용자의 주의가 계속 돌아가는 지점

출력 목표:

- 표면 기록 뒤에 있는 패턴을 식별한다.
- 관찰된 사실과 해석을 구분한다.
- "무슨 일이 있었는가"에서 "이 움직임이 무엇을 시사하는가"로 넘어간다.

금지:

- 극적인 추리소설식 표현
- 확정적인 심리 진단
- 단서를 증거처럼 단정하는 것

### 04 상황 추리

역할:
현재 상황이 감정적으로 어떻게 작동하고 있는지 추론한다. 사용자의 상태, 상대 또는
맥락, 그리고 풀리지 않은 긴장 사이의 연결을 다룬다.

입력에서 읽어야 하는 것:

- 왜 이 상황을 떠나거나 결정하거나 단순화하기 어려운지
- 어떤 감정적 욕구가 작동하고 있는지
- 이 흐름이 계속 반복되는 이유

출력 목표:

- 감정 상황의 구조를 설명한다.
- 왜 이 의뢰가 쉽게 정리되지 않는지 보여준다.
- 추론은 잠정적이고 관찰 가능한 수준으로 유지한다.

금지:

- 최종 판정을 내리는 것
- 사용자가 해야 할 행동을 지시하는 것
- 궁합 분석이나 미래 예측처럼 보이게 만드는 것

### 05 핵심 추리

역할:
리포트의 가장 밀도 높은 해석을 전달한다. 앞선 장면보다 짧고, 설명은 줄이며, 의미의
농도는 높아야 한다.

입력에서 읽어야 하는 것:

- 가장 중요한 감정 패턴
- 사용자의 표면 질문 아래에 있는 숨은 질문
- 사용자의 행동과 두려움이 만나는 지점

출력 목표:

- 리포트의 핵심 해석을 명명한다.
- 문장 수를 줄이고 의미 밀도를 높인다.
- 파일의 중심 판단처럼 느껴지게 한다.

금지:

- 앞선 관찰을 반복하는 것
- 새로운 갈래를 과도하게 추가하는 것
- 지나친 위로로 핵심 판단을 흐리는 것

### 06 최종 보고

역할:
완벽한 답을 내려주는 대신, 지금까지 열린 파일을 조용히 닫는다.

입력에서 읽어야 하는 것:

- 사용자가 이후에도 관찰해야 할 지점
- 지금까지 정리된 감정의 움직임
- 아직 해결되지는 않았지만 형태가 생긴 문제

출력 목표:

- 명확한 최종 보고의 톤을 남긴다.
- 명령이 아니라 관찰 포인트를 제공한다.
- 닫힘과 이어짐이 함께 느껴지게 한다.

금지:

- 일반적인 요약문으로 끝내는 것
- 동기부여 문장으로 마무리하는 것
- 감정적 해결이나 명확성을 과하게 약속하는 것

## 5. 장면별 입력 및 출력 목표

각 장면은 입력을 그대로 다시 말하는 것이 아니라, 입력을 리포트의 관찰 단위로
변환해야 한다.

입력에서 읽어야 하는 것:

- 감정의 방향
- 반복과 회피
- 말한 욕구와 실제 행동 사이의 모순
- 불확실성 앞에서 사용자가 서 있는 위치

출력이 제공해야 하는 것:

- 구체적인 관찰
- 근거 있는 추론
- 통제된 감정적 거리감
- 다음 장면을 열어볼 이유

## 6. 장면 공통 금지사항

모든 장면에서 아래 요소를 피한다.

- 직접적인 진단
- 치료적 주장
- 도덕적 판단
- 미래 예측
- 일반적인 연애 조언
- 과도한 위로
- "~해야 한다" 식의 지시
- 긴 설명형 에세이
- 채팅처럼 오가는 문장
- 사용자의 입력을 분량 채우기용으로 반복하는 것

## 7. 입력 부족 시 대응 방식

입력이 짧거나 모호하거나 불완전해도, 리포트는 관찰 범위를 좁혀 유효한 파일을
작성해야 한다.

사용 가능한 표현:

- "현재 기록만 보면..."
- "~쪽으로 기울어져 있는 것으로 보임"
- "~가능성이 있음"
- "~단정하기는 어렵지만..."

사용하지 않는 표현:

- "정보가 부족합니다."
- "더 자세히 알려주세요."
- "정확한 분석이 어렵습니다."

불확실성은 파일 경험을 깨는 방식으로 안내하지 않는다. 대신 문체와 추론의 강도를
조절해 표현한다.

## 8. 문체 가이드

문체는 조용한 보고서에 가까워야 한다.

권장 종결 표현:

- ~것으로 보임
- ~상태였음
- ~경향이 있음
- ~가능성이 있음
- ~흐름이 관찰됨
- ~쪽으로 기울어져 있음

문체 규칙:

- 문장은 정확하고 절제되어야 한다.
- 감정적 과장보다 차분한 추론을 사용한다.
- 관찰 동사를 우선한다: 보임, 관찰됨, 드러남, 반복됨, 기울어짐.
- "괜찮아요", "힘들었겠어요", "해보세요" 같은 상담 문구는 피한다.
- "나를 사랑하세요", "용기를 내세요" 같은 자기계발식 표현은 피한다.
- "분석 결과", "데이터에 따르면", "사용자님" 같은 AI 서비스 문구는 피한다.

## 9. Block 분류 기준

리포트는 채팅 말풍선이 아니라 문서 조각으로 구성되어야 한다.

권장 block 유형:

- 핵심 관찰: 장면을 여는 중심 문장
- 관찰 기록: 감정 행동에서 보이는 것
- 감정 메모: 행동 아래에서 움직이는 내면 상태
- 추론 조각: 패턴이 시사하는 것
- 종료 메모: 다음 장면으로 이어지는 관찰 지점

block 규칙:

- 각 block은 분명한 역할을 가져야 한다.
- 긴 연속 문단을 피한다.
- 질문과 답변이 오가는 리듬을 피한다.
- 모든 block이 같은 무게로 보이지 않게 한다.
- 후반 장면일수록 더 짧고 밀도 높은 block을 사용한다.

## 10. 심리탐정 톤 가이드

"심리탐정"은 역할극이 아니라 구조화 방식이다.

이 톤은 아래 목적에 사용한다.

- 모호함을 정리한다.
- 감정적 증거를 추적한다.
- 관찰과 추론을 분리한다.
- 사용자가 파일을 열람하고 있다는 느낌을 만든다.

아래 용도로 사용하지 않는다.

- 극적인 미스터리 서사 추가
- 캐릭터 대사 생성
- 범죄 현장, 누아르, 탐정 소설 클리셰 사용
- 게임 UI처럼 보이는 표현
- 사건, 용의자, 증거, 단서, 수사 같은 단어의 과도한 반복

감정 사건이라는 은유는 조용하게 유지되어야 한다. 사용자는 자신의 기록이 연출되고
있다고 느끼기보다, 신중하게 다뤄지고 있다고 느껴야 한다.

## 11. Anti-Pattern

AI 상담처럼 보이는 표현을 피한다.

- "당신은 지금 많이 힘든 상태입니다."
- "이럴 때는 먼저 자신을 돌보는 것이 중요합니다."
- "상대와 솔직하게 대화해보세요."

블로그나 에세이 구조를 피한다.

- 긴 도입문
- 번호가 붙은 조언
- 보편적 교훈
- 일반 심리학 설명

반복을 피한다.

- 사용자의 상황을 변환 없이 다시 말하는 것
- 모든 입력 답변을 요약하는 것
- 명백한 재진술로 분량을 채우는 것

과도한 해결을 피한다.

- 정답을 선언하는 것
- 상대의 진심을 예측하는 것
- 사용자가 특정 행동을 선택해야 한다고 결론 내리는 것

## 12. 장면 리듬 및 호흡

장면의 리듬은 정보량보다 중요하다. 리포트는 사용자가 기록을 순서대로 열어보는
경험이어야 하며, 각 장면은 거리감, 밀도, 감정 압력이 달라야 한다.

리듬 원칙:

- 각 장면은 서로 다른 읽기 온도를 가져야 한다.
- 앞 장면은 더 접근 가능하고 가까울 수 있다.
- 중간 장면은 더 관찰자적이고 구조적이어야 한다.
- 후반 장면은 설명을 줄이고 밀도를 높인다.
- 모든 장면을 같은 길이와 같은 무게로 작성해 사용자를 지치게 만들지 않는다.

장면별 리듬:

- 01 의뢰 접수: 가장 인간적이고 가까운 거리감. 파일이 조심스럽게 열렸다는 느낌을
  준다.
- 02 현장 기록: 설명적이고 안정적인 호흡. 눈에 보이는 상황을 정리한다.
- 03 단서 분석: 가장 관찰자에 가까운 시점. 거리감과 패턴을 만든다.
- 04 상황 추리: 연결과 설명의 장면. 감정 루프가 작동하는 방식을 보여준다.
- 05 핵심 추리: 가장 짧고 밀도 높은 장면. 리포트의 중심 판단처럼 느껴져야 한다.
- 06 최종 보고: 조용히 닫되 과하게 종결하지 않는다. 정답보다 이후 관찰 포인트를
  남긴다.

## 13. 장면 전환 규칙

각 장면은 다음 기록을 열어볼 필요가 느껴지는 방식으로 끝나야 한다.

전환 원칙:

- 마지막 문장은 전체 요약보다 다음 관찰 포인트에 가까워야 한다.
- 다음 내용을 과장되게 숨기는 방식으로 끝내지 않는다.
- 감정 흐름이 다음 장면으로 자연스럽게 이어지게 한다.
- 모든 장면을 리포트의 끝처럼 닫지 않는다.
- 마지막 문장은 조용하지만 방향성을 가져야 한다.

좋은 전환 형태:

- "이 지점에서 다음으로 봐야 할 것은 감정의 크기가 아니라, 그 감정이 반복되는 방식임."
- "따라서 다음 기록에서는 이 관계가 왜 쉽게 정리되지 않는지 확인할 필요가 있음."
- "문제는 선택 자체보다, 선택 앞에서 멈추게 되는 기준에 가까움."

피해야 할 전환:

- "충격적인 진실은 다음 장면에서 밝혀집니다."
- "이제 모든 답이 드러납니다."
- "결론은 명확합니다."

## 14. 향후 확장 기준

이 구조는 VEIL의 주요 카테고리 전반에 적용 가능해야 한다.

- 연애·결혼
- 인간관계
- 직업·진로
- 감정

6개 장면 구조는 안정적으로 유지한다. 카테고리별 차이는 새로운 리포트 구조를 만드는
방식이 아니라, focus, 감정 어휘, 관찰 패턴의 차이로 표현한다.

향후 프롬프트 작업은 이 문서를 방향 기준으로 삼을 수 있다. 단, 이 문서는 프롬프트
템플릿, 구현 세부사항, 시스템 동작을 정의하지 않는다.
